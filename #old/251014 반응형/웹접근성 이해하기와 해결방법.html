웹접근성

웹접근성이란? 모든 사용자가 신체적, 환경적 조건에 관계없이 웹에 접근하여 이용할 수 있도록 보장하는 것

웹사이트에서 제공하는 정보를 차별 및 제한없이 동등하게 이용할 수 있도록 보장하는 것

장애인 및 노약자, 고령자, 임산부, 다양한 플랫폼 또는 디바이스와 웹브라우저 등 모든 환경

1. 웹접근성 준수시 기대효과

  • 장애인, 고령자 등을 포함한 사용자층 확대
  • 규정과 법적 요구 사항에 대한 준수
  • 다양한 환경, 새로운 기기에서의 이용
  • 개발 및 운영의 효율성 제고
  • 사회 공헌 및 복지 기업으로서의 기업 이미지 향상

2. 웹콘텐츠 지침 4가지 항목 총 24개 세부항목에 따른 해결방법

1) 인식의 용이성(7)
1. 대체 텍스트 : 텍스트 아닌 콘텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 한다. => img태그 사용시 alt속성의 값을 넣어준다. 의미가 없을 경우는 생략한다.
2. 멀티미디어 대체 수단 : 영상 콘텐츠 삽입시 자막을 삽입한다. => vtt파일
3. 명료성 : 콘텐츠는 색에 관계없이 인식될 수 있어야 한다. => 그래프, 챠트, 지도, 필수입력 등과 같은 항목에 색상만으로 구분하지 않도록 한다. => 크기, 밑줄, 기울기 스타일로 해결
4. 명료성 : 지시 사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 한다. 명확한 지시사항을 표시해야 한다. => *기호 설명 따로 작성, 기호는 엔티티 코드로 작성
5. 명료성 : 텍스트 콘텐츠와 배경 간의 명도 대비는 4.5대 1 이상이어야 한다. => color contrast 도구 체크
6. 명료성 : 자동으로 소리가 재생되지 않아야 한다.(영상, 음악) => 자동재생 안되게 하고, 자동으로 하려면 반드시 muted 설정한다.
7. 명료성 : (콘텐츠 간의 구분) 이웃한 콘텐츠는 구별될 수 있어야 한다. => 테두리, 색상, 밑줄, 간격 등으로 구분한다.
2) 운용의 용이성(9)
1. 모든 기능은 키보드만으로도 사용할 수 있어야 한다. => tab키를 사용하여 이동, enter키 실행하여 테스트
2. 키보드에 의한 초점은 논리적으로 이동해야 하며 시각적으로 구별할 수 있어야 한다. => 키보드 tab키 사용시 테두리 표시하기
3. 사용자 입력 및 컨트롤은 조작 가능하도록 제공되어야 한다. => 버튼의 크기를 최소 6mm이상(25px)설정 또는 패딩값으로 여백(19px)을 설정하여 클릭을 쉽게 할 수 있도록 한다.
4. 시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 한다. => 시간제한이 있는 콘텐츠는 연장할 수 있는 메뉴를 추가하여 사용자가 선택할 수 있게 한다.(예) 로그인 시간연장, 인증번호 시간연장
5. 자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 한다. => 페이지 자동전환시 사용자가 멈추거나 넘길수 있는 'stop', 'play'버튼 추가(메인 슬라이드, 이미지 슬라이드도 포함)
6. 초당 3~50회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 한다. => 노약자, 장애인, 임신부 등이 놀랄 수 있기 때문에 사용하지 않는다.(사이트 참고 https://d-chris.com)
7. 콘텐츠의 반복되는 영역은 건너뛸 수 있어야 한다. => href속성값을 확인, skip, 바로가기 내비게이션 추가
8. 페이지, 프레임, 콘텐츠 블록(section, article)에는 적절한 제목을 제공해야 한다. => title태그에 제목 작성, section, article작성시 제목요소 추가
9. 링크 텍스트(a)는 용도나 목적을 이해 할 수 있도록 제공해야 한다. => a태그 사용시 title속성값 반드시 작성
3) 이해의 용이성(6)
1. 주로 사용하는 언어를 명시해야 한다. => 문서의 첫줄에 언어 명시 html lang="ko"
2. 사용자가 의도하지 않은 기능(새창, 초점에 의한 맥락 변화 등)은 실행되지 않아야 한다. => (새창, 초점에 의한 맥락변화, 모달윈도) * 새로운 윈도우 창이 뜨지 않게 할 것, 화면을 가리지 말것, 배경색을 투명하게 하여 모달윈도를 띄우거나, 페이지 상단에 배너를 삽입해야 함. (예-알톤스포츠)
3. 콘텐츠는 논리적인 순서로 제공해야 한다. => 제목은 h2, h3.... 내용은 p태그로 작성해야 더보기 버튼은 태그 내용 아래에 작성(디자인에서 위에 있다고 먼저 작성하면 안됨.)
4. 표(table)는 이해하기 쉽게 구성해야 한다. => table태그 작성시 caption, 제목셀은 th, 내용셀은 td, thead, tbody태그로 영역을 설정하여 작성, summary(요약), scope로 읽어주는 방향 설정 함.
5. 사용자 입력(input)에는 대응하는 레이블(label)을 제공해야 한다. => radio, checkbox태그 사용시 label태그(제목) 사용
6. 입력 오류를 정정할 수 있는 방법을 제공해야 한다. => '다시쓰기', '취소하기', 'X'버튼을 만든다. 안내문구를 추가(빨강으로 강조) input 태그 사용시 placeholder속성을 사용한다.
4) 견고성(2)
1. 문법 준수 => html마크업 검사, css유효성 검사 테스트
2. 웹 애플리케이션 접근성 준수 => * 콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 * 텍스트, 이미지, 사운드, 영상 콘텐츠를 사용하여 작성 웹접근성 테스트 : 구글 크롬브라우저 개발자모드 'Light House"에서 테스트
